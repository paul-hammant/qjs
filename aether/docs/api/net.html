http_response_status

http_response_status(response)

Response field accessors. All are NULL-safe: passing NULL or a freed response returns a sensible default (0 or "") rather than crashing. Returned const char* pointers are owned by the response struct and valid until http_response_free() is called.

http_response_ok

http_response_ok(response)

Convenience: returns 1 if the request succeeded (no transport error AND HTTP status is in the 2xx range), 0 otherwise. Use this for the common "did it work?" check instead of chaining error/status calls.

http_response_status_code

http_response_status_code(response)

Legacy accessor aliases kept for callers that used the older `_code` / `_str` names. Prefer the short names above.
